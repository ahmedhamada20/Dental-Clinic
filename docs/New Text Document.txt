Client Project Description
(Description for Flutter Developer)
Project Overview

The project is a Dental Clinic Management System consisting of two main components:

Laravel Dashboard + REST API

Flutter Mobile Application for Patients

The purpose of the system is to digitize the workflow of a dental clinic, including:

Patient management

Appointment scheduling

Visit management

Dental chart

Treatment plans

Medical files

Invoices and payments

Notifications

Reporting

Currently, the clinic has one branch and one doctor, but the system should be designed in a way that allows expansion in the future.

System Components
1. Admin Dashboard (Laravel)

The dashboard will be used by:

Admin

Doctor

Receptionist

Assistant

The dashboard manages:

Patients

Appointments

Visits

Dental chart

Treatment plans

Services

Invoices

Payments

Promotions

Medical files

Reports

Notifications

2. Mobile Application (Flutter)

The mobile app is only for patients.

The main goal of the mobile app is to allow patients to:

Create an account

Book appointments

Cancel appointments

Receive notifications

View invoices

View treatment plans

View dental files and X-rays

View ticket number for clinic entry

Patient Authentication

Patients will register using:

Phone number

Password

Authentication will be handled through the Laravel API.

Appointment Booking System

Patients can book appointments from the mobile app.

Booking Flow

Patient selects a service

Patient selects date

Patient selects available time slot

Each service has a specific duration, so the system calculates available slots dynamically.

Appointment Rules

Two patients cannot book the same time.

Booking must be done before the appointment time.

Service duration determines slot availability.

Waiting List Feature

If no appointments are available:

The patient can join a waiting list.

If another patient cancels their appointment:

The system will notify the first patient in the waiting list to complete booking.

Appointment States

Appointments have the following statuses:

Pending

Checked In

In Progress

Completed

Cancelled by Patient

Cancelled by Clinic

No Show

Check-in Process

When the patient arrives at the clinic:

The receptionist performs Check-in from the dashboard.

At this moment:

The appointment becomes a Visit

A Ticket Number is generated.

Ticket System

Each patient receives a ticket number.

Example:

Ticket 23

Rules:

Ticket numbers reset daily.

The ticket is displayed in the mobile app.

Used for patient order inside the clinic.

Visit Process

During the visit, the doctor can:

Write diagnosis

Update dental chart

Add performed services

Upload files or X-rays

Create treatment plan

Add notes

Dental Chart

The system includes a full dental chart (odontogram).

It contains:

32 teeth

Upper and lower jaw

Tooth status tracking

Possible tooth states include:

Healthy

Caries

Filling

Root Canal

Crown

Implant

Extracted

Bridge

Only the doctor should modify the dental chart.

Treatment Plan

The doctor can create treatment plans.

Example:

Root Canal Treatment

Sessions:

Session 1

Session 2

Session 3

Patients will be able to see their treatment plan inside the mobile app.

Medical Files

The doctor can upload:

X-rays

Prescription

Treatment files

Before/after photos

Lab results

Patients can view all uploaded files in the mobile app.

Invoice System

Invoices are generated after the visit.

Because the actual treatment may differ from the booked service.

Example:

Patient booked cleaning.

Doctor discovered root canal was required.

Invoice is generated based on actual performed services.

Payments

The system supports multiple payment methods.

Payment methods include:

Cash

Bank Transfer

Instapay

Vodafone Cash

Mixed payments are allowed.

Example:

500 Cash
500 Instapay

Discounts & Promotions

The system supports temporary promotions.

Examples:

Free consultation for 3 days

20% discount on cleaning

Fixed discount on visits

Promotions can have:

Start date

End date

Notifications
